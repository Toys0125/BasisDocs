---
title: Ограничение использования CPU в Unity
---

При сборке проекта в Unity 6 он будет использовать ВСЕ системные ресурсы, делая систему полностью непригодной для использования (особенно во время этапа компиляции шейдеров), даже на высокопроизводительном компьютере.

Вы можете использовать следующий аргумент для лаунчера, чтобы предотвратить блокировку каждого ядра CPU на 100%.

```
-job-worker-count 8
```

Где **8** должно быть числом меньше общего количества реальных физических ядер CPU вашей машины. Это ограничит дочерние воркеры Unity данным значением, тем самым не позволяя блокировать каждое ядро CPU. Это даже может улучшить скорость сборки Unity, поскольку оно, как правило, оказывается чрезмерно нагруженным и замедляет само себя.
