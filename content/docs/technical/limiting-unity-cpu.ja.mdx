---
title: UnityのCPU使用量を制限する
---

Unity 6でプロジェクトをビルドする際、ハイエンドマシンでも全システムリソースを使い尽くし、完全に使用不能になる傾向があります（特にシェーダーコンパイル段階）。

以下の引数をランチャーに使用することで、すべてのCPUコアを100%にロックされないようにすることができます。

```
-job-worker-count 8
```

ここで **8** はマシンの実際の物理CPUコア総数より少ない数にしてください。これにより、Unityの子ワーカーがこの値に制限され、すべてのCPUコアのロックアップが防止されます。これにより、Unityが過負荷状態になり自身を低速化させる傾向があるため、ビルド速度が向上する場合もあります。

プロジェクトの起動引数にアクセスするには、Unity Hubを開いてプロジェクト名を右クリックします：
