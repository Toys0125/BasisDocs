---
title: 限制 Unity CPU 使用率
---

在 Unity 6 上构建项目时，它往往会使用您系统的所有资源，即使在高端机器上也会导致系统完全无法使用（尤其是在着色器编译阶段）。

您可以将以下参数用作启动器参数，以阻止其将每个 CPU 核心锁定在 100%。

```
-job-worker-count 8
```

其中 **8** 应该是小于您机器实际物理 CPU 核心总数的数字。这将把 Unity 子工作进程限制在此值，从而防止其锁定每个 CPU 核心。这甚至可能提高 Unity 的构建速度，因为它往往会过度竞争并使自身减速。

您可以通过打开 Unity Hub 并右键单击项目名称来访问项目的启动参数：
