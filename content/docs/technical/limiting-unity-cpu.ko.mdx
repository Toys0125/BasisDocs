---
title: Unity CPU 사용량 제한
---

Unity 6에서 프로젝트를 빌드할 때 고사양 머신에서도 모든 시스템 리소스를 사용하여 완전히 사용 불가능한 상태로 만드는 경향이 있습니다 (특히 셰이더 컴파일 단계에서).

런처의 인수로 다음을 사용하면 모든 CPU 코어가 100%로 잠기는 것을 방지할 수 있습니다.

```
-job-worker-count 8
```

여기서 **8**은 머신의 실제 물리적 CPU 코어 총 수보다 적은 숫자여야 합니다. 이렇게 하면 Unity 자식 워커가 이 값으로 제한되어 모든 CPU 코어가 잠기는 것을 방지합니다. 이렇게 하면 Unity 빌드 속도가 향상될 수도 있습니다. 과도한 경합으로 인해 스스로 느려지는 경향이 있기 때문입니다.

Unity Hub를 열고 프로젝트 이름을 우클릭하여 프로젝트의 실행 인수에 접근할 수 있습니다:

![Unity Hub 프로젝트 우클릭 메뉴](/img/limiting-unity-cpu/1.png)
