## Häufig gestellte Fragen

### Was ist BasisVR?

BasisVR ist ein Software-Framework, das Entwickler mit tiefen Technologieintegrationen und erstklassigen Ansätzen ausstattet, um soziale Erlebnisse in Virtual Reality zu schaffen.

<Callout type="info">
Hinweis: ***Basis ist keine Social-VR-Plattform***. Es enthält nützliche Primitive, aber das Kernprojekt ist nicht dafür gedacht, die Art von skalierbarer sozialer Infrastruktur zu schaffen, die Social-VR-Plattformen benötigen. Es hindert Ersteller, die ihre eigene erstellen möchten, nicht daran, aber das Projekt selbst liefert Frameworks und Demos.
</Callout>

### Was sind die Systemanforderungen?

BasisVR erfordert:
- Einen VR-fähigen PC oder ein eigenständiges VR-Headset
- Unity für die Entwicklung
- Kompatible VR-Controller

### Wie fange ich an?

1. BasisVR installieren
2. Avatar einrichten
3. Eine Welt erstellen oder beitreten
4. Mit dem Erkunden beginnen!

Schauen Sie sich die [Einführung](/docs) für detaillierte Einrichtungsanweisungen an.

### Kann ich meine eigenen Avatare erstellen?

Ja! BasisVR unterstützt benutzerdefinierte Avatare. Weitere Informationen zum Erstellen und Importieren benutzerdefinierter Avatare finden Sie im [Avatar-Leitfaden](/de/docs/avatar).

### Wie erstelle ich meine eigene Welt?

Sie können mit Unity benutzerdefinierte Welten erstellen. Schauen Sie sich den [Welt-Erstellungsleitfaden](/de/docs/world) für schrittweise Anweisungen an.

### Wo bekomme ich Hilfe?

- Diese Dokumentation konsultieren
- Unsere Community-Foren besuchen
- Probleme auf GitHub einreichen
- Unserem Discord-Server beitreten

### Wie kann ich beitragen?

Wir begrüßen Beiträge! Weitere Informationen zum Mitmachen finden Sie im [Beitragsleitfaden](/de/docs/contributing).

### Was ist der Unterschied zwischen Kernkomponenten-Beiträgen und Add-on-Beiträgen?
A: Kernkomponenten umfassen Systeme, die die Hauptfunktionen von Basis ausmachen. Beiträge (oder Module) umfassen zusätzliche Funktionen, die zwar nützlich, aber standardmäßig nicht erforderlich sind. Daher können Beiträge Funktionen bereitstellen, die außerhalb des Hauptangebots von Basis liegen und für Entwickler, die Basis verwenden, wertvoll sein können.

### Gefährdet das Teilen von BEE-Dateien die Anmeldedaten des Remote-Content-Hosting-Servers?

Nein! Sie können Ihre BEE-Dateien beispielsweise mit Ihren Freunden teilen. Eine BEE-Datei umfasst den Basis-Asset (auch für mehrere Plattformen), mit Ausnahme des Verschlüsselungsschlüssels zur Entschlüsselung. Mit anderen Worten, es ist im Wesentlichen ein Multi-Asset-Bundle. Wenn Sie das Passwort einer BEE-Datei teilen, kann jeder, der es hat, auf alles zugreifen, was sich in der BEE-Datei befindet (d. h. den spezifischen UGC wie Requisiten, Avatare, Welten).

## Wie kann ich von verschiedenen Geräten aus eine Verbindung zu einem lokalen Server herstellen?

Es gibt mehrere Möglichkeiten, von verschiedenen Geräten aus eine Verbindung zu einem lokalen Server herzustellen. Sie können beispielsweise die Portweiterleitung auf Ihrem Router aktivieren. Wenn Sie Ihren Router lieber nicht ändern möchten, können Sie alternativ [`Tailscale`](https://tailscale.com/download) installieren, das sehr einfach einzurichten ist.

## Wo kann ich einen Remote-Server einrichten?

Es gibt eine Vielzahl von Cloud-Diensten, auf denen Sie Ihre Server hosten können. Beispiele: Bunny, DigitalOcean, CloudFlare, AWS, Microsoft Azure, Google Cloud und mehr.
