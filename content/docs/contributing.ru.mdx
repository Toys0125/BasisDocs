## Руководство по вкладу в BasisDocs

Эта страница создана для помощи тем, кто хочет внести вклад в документацию репозитория BasisDocs.

BasisDocs использует Fumadocs, современный фреймворк для документации. Основные директории, с которыми вы будете работать:

- `content/docs/` - Содержимое документации в формате MDX
- `public/img/` - Статические изображения и ресурсы

![Структура документации](/img/basisdocs-contrib/1.png)

## Начало работы

Быстрее всего освоиться на этом сайте, познакомившись с:

- [Документацией MDX](https://mdxjs.com/) - Расширенный Markdown с JSX-компонентами
- [Документацией Fumadocs](https://fumadocs.vercel.app/) - Фреймворк документации, который мы используем

### Доступные компоненты

В своей документации вы можете использовать различные компоненты:

- `<Callout type="info|warn|error">` - Для важных заметок и предупреждений
- `<Card>` и `<Cards>` - Для навигационных карточек
- Блоки кода с подсветкой синтаксиса
- Изображения с соответствующим alt-текстом

## Возможности Markdown

Поддерживается стандартный Markdown, а также расширенные функции:

- **Таблицы** - Для структурированных данных
- **Блоки кода** - С подсветкой синтаксиса для конкретного языка
- **Admonitions** - С использованием компонентов Callout
- **Ссылки** - Внутренние и внешние
- **Изображения** - Правильно ссылающиеся из `/public/img/`

## Внесение вклада в документацию

1. **Форкните репозиторий**
   
   Форкните [репозиторий BasisDocs](https://github.com/BasisVR/BasisDocs) на GitHub.

2. **Клонируйте свой форк**
   
   ```bash
   git clone https://github.com/YOUR_USERNAME/BasisDocs.git
   cd basis-docs
   ```

3. **Установите зависимости**
   
   ```bash
   npm install
   ```

4. **Запустите сервер разработки**
   
   ```bash
   npm run dev
   ```

5. **Внесите изменения**
   
   Редактируйте MDX-файлы в `content/docs/` и добавляйте изображения в `public/img/`.

6. **Протестируйте изменения**
   
   ```bash
   npm run build
   ```

7. **Отправьте Pull Request**
   
   Отправьте изменения и откройте pull request на GitHub.

## Рекомендации

- Используйте описательные заголовки и правильную иерархию
- Добавляйте alt-текст ко всем изображениям
- Используйте компоненты Callout вместо простых текстовых предупреждений
- Следуйте существующей структуре файлов и соглашениям об именовании
- Тестируйте изменения локально перед отправкой

Заранее спасибо за ваш вклад!
