## Unityエディターのセットアップ

- 最新のサポートされているUnityエディターバージョンのインストールとBasisフレームワークの取得については、[はじめに](/docs#installing-unity)セクションを参照してください。
- Unity Hubを開き、既存のプロジェクトを追加して、展開/クローンしたBasisプロジェクトフォルダーを選択します
- プロジェクトの初回読み込み時に以下のプロンプトが表示されますので、「Restart Editor」をクリックしてください。

![Unity再起動プロンプト](/img/avatars/0.png)

- プロジェクトが読み込まれたら、アバターファイルを読み込むことができます：
- いずれか、
  - プロジェクト階層で`Assets`を見つけ、`Assets`内の空のスペースを右クリックして`Show in Explorer`を選択する
  - アバターファイル [メッシュ/マテリアル/テクスチャなど] をコピーする
- または
  - AssetsメニューからImport Package > Custom Packageを使用して、ファイルを含む.unityパッケージをインポートする
- Unityウィンドウに戻り、インポートプロセスが完了するまで待ちます。

<Callout type="info">
コンテンツを新しいディレクトリ「Assets > _UserContent」に配置すると、プロジェクトのデフォルト.gitignoreファイルがこのフォルダーのgitの変更を非表示にします。
</Callout>

## モデルのインポート

- アバターモデルの場所を見つけます（通常は.FBX形式で提供されます）。
- ファイルをクリックして、Import Settings ウィンドウの`Model`セクションに移動します。
- 以下のように`Red/Write`と`Legacy Blend Shape Normals`が選択されていることを確認します。

![モデルインポート設定](/img/avatars/1.png)

- Import Settings ウィンドウの`Rig`セクションに移動します。
- `Animation Type`ドロップダウンメニューをクリックして`Humanoid`を選択し、以下のように`Apply`を押します。

![リグ設定](/img/avatars/2.png)

- その後、`Apply`の上にある`Configure`ボタンが選択可能になるので、クリックしてボーンマッピングを確認します。

![ボーンマッピング](/img/avatars/3.png)

- すべての必須ボーンが選択されていることが確認できるはずです。アーマチュアに対して正しく見えることを確認してください。
- 最後に、`Mapping`ウィンドウの`Head`ボタンをクリックして、`Jaw`の割り当てを削除します。これは通常誤った設定です。

![頭部マッピング](/img/avatars/4.png)

<Callout type="info">
場合によっては、Unityインスペクターがアバターのボーンマッピングが適切に割り当てられていない、またはアバターが適切にTポーズになっていないと言及することがあります。これを修正するには、MappingとPoseのドロップダウンをクリックして、AutomapまたはEnforce T-Poseを選択できます。

![アバターリグのオートマップ方法を示すインスペクターウィンドウ](/img/avatars/4a.png)

![Tポーズの強制方法を示すインスペクターウィンドウ](/img/avatars/4b.png)

</Callout>

## スキンドメッシュのセットアップ

- 現在、アバターのスキンドメッシュを仮想プラットフォームで使われる`Body`などの名前にする必要はありません。
- 複数のスキンドメッシュがある場合は、シーン内のライトプローブを継承するために、すべて同じ参照点を使用するようにピン留めすることを検討してください。これは、アーマチュアのボーン（通常はHips）を`Anchor Override`フィールドに割り当てることで行えます。

![アンカーオーバーライド](/img/avatars/5.png)

- 複数のスキンドメッシュを持つアバターのもう一つの考慮点は、異なるバウンディングボックスサイズにより、アバターの残りの部分がリモートビューワーに見えている間に個々のメッシュが表示/非表示になることです。
- Skinned Mesh RendererコンポーネントのトップにあるBoundsというセクションで、Center X、Y、Z と Extent X、Y、Zの値を上書きできます。通常、別々のスキンドメッシュ間でこれを同じ値に設定するのが良いでしょう。

![バウンド設定](/img/avatars/6.png)

## マテリアル/シェーダー

- Unity 6に付属のUniversal Render Pipeline/Litシェーダーをすぐに使い始めることができます
- または、使用できる互換性のある[アバターシェーダー](/ja/docs/avatar/shaders)のリストをまとめています。

## Basis Avatarのセットアップ

- 上記の手順により、アバタープレファブをシーンに引き込んだときに自動的にAnimatorコンポーネントが追加されるはずです。
- アバターの親GameObjectで`Add Component`をクリックし、`Basis Avatar`を追加します。
- 希望するアバター名と説明を入力します。
- `Assigned Input`で、Animatorが割り当てられていることを確認します。通常、これは自動的に行われます。また、アバターの顔とまばたきのブレンドシェイプを含むSkinned Mesh Rendererを割り当てます。オプションで、アバターのサムネイルアイコンを追加することもできます。
- `Viseme Setup`で、アバターの関連するビジームとまばたきを割り当てます。自動検出ボタンを使用して自動的に行うこともできます。
- オプションで、プレイモードに入ってから`Test in Editor`オプションを選択することで、エディターでアバターをテストできます。
- `Create Avatar Bee File`を選択する前に、すべてのシェーダーが互換性があることを確認してください。アバターに互換性のないシェーダーエラーがある場合（衣服やアクセサリーを含む）、BEEファイルは生成されません。問題がある場合、インスペクターが報告しますので、必ず解決してください！

![Basis Avatarコンポーネントが追加されたインスペクターウィンドウ](/img/avatars/6a.png)

![アバターブレンドシェイプの動きの例を示すインスペクターウィンドウ](/img/avatars/6b.png)

<Callout type="info">
Skinned Mesh Rendererのブレンドシェイプやアバターのアップロードが初めての方のために説明すると、基本的にはアバターのモーフを変更するために制御できるスライダーです。

![アバターブレンドシェイプの動きの例を示すインスペクターウィンドウ](/img/avatars/6c.gif)
</Callout>

<Callout type="type">
注意：`Basis Avatar`のブレンドシェイプがすべてなくてもアバターをアップロードできます。また、目のブレンドシェイプのみがサポートされており、目のボーンは現在サポートされていません。
</Callout>

## コンストレイントのセットアップ

BasisはUnity内蔵のConstraintsシステムを使用せず、代わりにAnimation Riggingを使用します。セットアップガイドは[こちら](/ja/docs/avatar/animation-rigging#setup-for-animation-rigging-packages)をご覧ください。
