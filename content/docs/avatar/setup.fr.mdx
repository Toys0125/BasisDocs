## Configurer l'éditeur Unity

- Conformément à la section [Démarrage](/docs#installing-unity) pour installer la dernière version de l'éditeur Unity prise en charge et acquérir le framework Basis.
- Ouvrez Unity Hub et ajoutez un projet existant, choisissez le dossier du projet Basis extrait/cloné
- Lors du chargement initial du projet, vous verrez l'invite suivante, veuillez cliquer sur 'Restart Editor'.

![Invite de redémarrage Unity](/img/avatars/0.png)

- Une fois le projet chargé, vous pouvez maintenant charger vos fichiers Avatar :
- Soit,
  - Trouvez `Assets` dans la hiérarchie du projet, à l'intérieur d'`Assets` faites un clic droit sur un espace vide et sélectionnez `Show in Explorer`
  - Copiez vos fichiers Avatar [Mesh/Matériaux/Textures, etc.]
- ou
  - Importez un package .unity contenant vos fichiers, depuis le menu Assets, Import Package > Custom Package
- Revenez à la fenêtre Unity et laissez le processus d'importation se terminer.

<Callout type="info">
Si vous placez votre contenu dans un nouveau répertoire "Assets > _UserContent", le fichier .gitignore par défaut du projet masquera les modifications git dans ce dossier.
</Callout>

## Importer le modèle

- Trouvez l'emplacement de votre modèle Avatar (généralement fourni au format .FBX).
- Cliquez sur le fichier puis naviguez vers la section `Model` de la fenêtre Import Settings.
- Assurez-vous que `Red/Write` et `Legacy Blend Shape Normals` sont sélectionnés comme indiqué ci-dessous.

![Paramètres d'importation du modèle](/img/avatars/1.png)

- Naviguez vers la section `Rig` de la fenêtre Import Settings.
- Cliquez sur le menu déroulant `Animation Type` et sélectionnez `Humanoid`, appuyez sur `Apply` comme indiqué ci-dessous.

![Paramètres du rig](/img/avatars/2.png)

- Ensuite, le bouton `Configure` au-dessus d'`Apply` est maintenant sélectionnable, cliquez dessus puis vérifiez vos mappages d'os.

![Mappages d'os](/img/avatars/3.png)

- Cela devrait montrer tous les os obligatoires comme sélectionnés ; assurez-vous que cela est correct pour votre Armature.
- Enfin, cliquez sur le bouton `Head` dans la fenêtre `Mapping` et supprimez l'assignation de la `Jaw` ; cela est généralement incorrect.

![Mappage de la tête](/img/avatars/4.png)

<Callout type="info">
Dans certains cas, l'inspecteur Unity peut mentionner que les mappages d'os de votre avatar ne sont pas correctement assignés, ou que l'avatar n'est pas correctement en T-pose. Pour corriger cela, vous pouvez cliquer sur les menus déroulants Mapping et Pose et sélectionner Automap ou Enforce T-Pose.

![Fenêtre inspecteur montrant comment automaquer le rig d'avatar](/img/avatars/4a.png)

![Fenêtre inspecteur montrant comment imposer la T-pose](/img/avatars/4b.png)

</Callout>

## Configuration du Skinned Mesh

- Il n'y a actuellement aucune exigence de nommer le Skinned Mesh de votre Avatar `Body` ou autre variante utilisée sur les plateformes virtuelles.
- Si vous avez plusieurs skinned meshes, envisagez de les épingler pour qu'ils utilisent tous le même point de référence sur le corps, pour hériter des sondes lumineuses dans la scène. Vous pouvez le faire en assignant un os de l'Armature (généralement Hips) au champ `Anchor Override`.

![Anchor override](/img/avatars/5.png)

- Une autre considération pour les avatars avec plusieurs Skinned Meshes est lorsque des tailles de boîte englobante différentes provoquent l'apparition et la disparition de maillages individuels pendant que le reste de l'Avatar est visible pour les téléspectateurs distants.
- Vous pouvez remplacer les valeurs Center X, Y, Z et Extent X, Y, Z dans la section nommée `Bounds` en haut du composant Skinned Mesh Renderer. Habituellement, c'est une bonne idée de définir cela identiquement pour les Skinned Meshes séparés.

![Paramètres de limites](/img/avatars/6.png)

## Configuration de Basis Avatar

- Les étapes ci-dessus devraient automatiquement ajouter un composant Animator à votre prefab Avatar lorsque vous le faites glisser dans la Scène.
- Sur le GameObject parent de l'Avatar, cliquez sur `Add Component` et ajoutez `Basis Avatar`.
- Entrez le nom et la description préférés de votre Avatar.
- Dans `Assigned Input`, assurez-vous qu'un Animator est assigné. Habituellement, cela sera fait automatiquement. De plus, assignez les Skinned Mesh Renderers contenant les blendshapes de visage et de clignement de votre avatar. Vous pouvez également ajouter une icône de miniature optionnelle pour votre avatar si vous le souhaitez.
- Dans `Viseme Setup`, assignez les visèmes et clignements pertinents pour votre avatar. Vous pouvez également utiliser les boutons de détection automatique qui essaieront de le faire automatiquement pour vous.
- Optionnellement, vous pouvez tester votre avatar dans l'éditeur en entrant d'abord en mode jeu puis en sélectionnant l'option `Test in Editor`.
- Avant de sélectionner `Create Avatar Bee File`, assurez-vous que tous vos shaders sont compatibles. Le fichier BEE ne se génèrera pas s'il y a des erreurs de shader incompatibles sur votre avatar (cela inclut les vêtements et les accessoires). S'il y a des problèmes, l'inspecteur les signalera, alors assurez-vous de les résoudre !

![Fenêtre inspecteur avec le composant Basis Avatar ajouté et les champs assignés](/img/avatars/6a.png)

![Fenêtre inspecteur montrant un exemple de mouvements de blendshape d'avatar](/img/avatars/6b.png)

<Callout type="info">
Si vous êtes nouveau avec les blendshapes du Skinned Mesh Renderer ou le téléversement d'avatars en général, ce sont essentiellement des curseurs que vous pouvez contrôler pour changer les morphes sur votre avatar.

![Fenêtre inspecteur montrant un exemple de mouvements de blendshape d'avatar](/img/avatars/6c.gif)
</Callout>

<Callout type="type">
Remarque : Vous pouvez quand même téléverser votre avatar même s'il n'a pas tous les blendshapes de `Basis Avatar`. De plus, notez que seuls les blendshapes oculaires sont pris en charge ; les os oculaires ne sont actuellement pas pris en charge.
</Callout>

## Configuration des contraintes

Basis n'utilise pas le système de contraintes Unity intégré, il utilise à la place Animation Rigging. Vous pouvez trouver le guide de configuration [ici](/fr/docs/avatar/animation-rigging#setup-for-animation-rigging-packages).
