## 소품 업로드

Basis를 사용하면 원격 URL 위치에서 소품 .BEE 파일을 로드할 수 있으며, 이는 네트워크 세션의 다른 사용자들도 에셋을 로드하기 위해 필요합니다.

<Callout type="info">
`https://BEE파일위치.BEE#(Base64Encode(Password))`를 사용하여 .BEE 파일 URL에 소품 비밀번호를 직접 추가할 수 있습니다.<br />
`#`과 비밀번호의 Base64 인코딩 문자열을 추가하는 것입니다.
</Callout>

이를 지원하려면 원격 위치에 직접 링크해야 합니다. 이 예제에서는 Google Drive를 사용합니다.

(Google Drive는 바이러스 검사 정책으로 인해 최대 100MB 파일의 직접 링크를 지원합니다.)

**BEE 파일**을 Google Drive 또는 로그인 없이 파일에 직접 링크할 수 있는 호스팅 공급자에 업로드합니다.

- 드라이브에 폴더를 만들고 공유 설정을 "링크가 있는 모든 사용자"로 설정합니다.

이제 이 폴더에 파일을 드래그하면 개별적으로 해당 권한을 설정할 필요가 없습니다.

- 방금 폴더에 업로드한 에셋의 오른쪽 상단에 있는 햄버거 메뉴를 클릭합니다.
- "공유"를 찾아 "링크 복사"를 클릭합니다.

BEE 파일에 필요한 정보를 얻기 위해 URL을 살펴봐야 합니다.

예:
```
https://drive.google.com/file/d/YourIDBringsAllTheLagToMyYard/view?usp=sharing
```

URL 내의 고유 식별자를 복사하여 다음에 추가합니다:
```
https://drive.google.com/uc?export=download&id=
```

예:
```
https://drive.google.com/uc?export=download&id=YourIDBringsAllTheLagToMyYard
```

<Callout type="warn">
**생성된 비밀번호 파일을 업로드하거나 공유하지 마세요**
</Callout>

## Basis에서 로드하기
<Steps>

<Step>
Basis를 실행하고, 메뉴를 열어 라이브러리를 선택합니다.

![Basis 메뉴](/img/props/8.png)
</Step>

<Step>
라이브러리 메뉴에서 창의 오른쪽 상단 근처에 있는 '새 콘텐츠 추가'를 선택합니다.

![새 콘텐츠 추가 버튼](/img/props/9.png)
</Step>

<Step>
**BEE 파일 URL**을 BEE 파일 URL 필드에 붙여넣고, **초극비 암호화 비밀번호**를 BEE 파일 비밀번호 필드에 입력한 후 아래의 '추가'를 클릭합니다.

![BEE 파일 URL 및 비밀번호](/img/props/10.png)
</Step>

<Step>
BEE 파일의 내용이 라이브러리에 추가되기 전에 유효성 검사가 수행됩니다.

![BEE 파일 추가 버튼](/img/props/11.png)
</Step>
</Steps>

## 소품 관리

라이브러리에서 최근에 추가된 소품을 찾을 수 있습니다.

![라이브러리의 새 소품](/img/props/12.png)

창 하단의 해당 버튼을 클릭하여 소품을 스폰, 삭제하거나 월드에서 다른 사람과 공유할 수 있습니다.

![소품 세부 정보](/img/props/13.png)

또한 여러 유용한 정보 패널을 볼 수 있습니다:

- **(1) 소품 정보** - 세부 정보 패널에는 컴파일 타임스탬프, 제목 및 설명이 표시됩니다.
- **(2) 플랫폼 호환성** - 아이콘은 이 소품을 지원하는 OS 플랫폼을 나타냅니다.
- **(3) 통계** - 삼각형, 재질 및 본 수를 표시합니다.
- **(4) 특성** - 소품의 네트워크 동작과 지속성을 설정할 수 있습니다.
- **(5) 핀** - 이 소품을 상단 메인 메뉴 창에 고정할 수 있습니다.

![스폰 바운딩 박스](/img/props/14.png)

'스폰' 버튼을 클릭하면 월드에서 소품을 스폰할 정확한 위치를 지정할 수 있습니다.

![스폰된 소품](/img/props/15.png)

잠시 후 소품 특성에 따라 에셋이 로컬 또는 네트워크로 나타납니다.
