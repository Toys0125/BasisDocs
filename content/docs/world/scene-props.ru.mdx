## Как появить проп с помощью кнопки сцены

Компонент Runtime Loader будет генерировать все пропы, расположенные в *Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0\initialresources*, во время выполнения. Также возможно использовать кнопку в сцене для создания пропов. Для этого:

- Добавьте объект canvas в вашу сцену. Убедитесь, что прикреплены соответствующие компоненты, аналогичные тем, что были показаны в предыдущих разделах. Не забудьте добавить компонент Basis Prop для создания .bee-файла для вашего пропа и его загрузки на удалённый сервер.

![Окно Inspector с canvas и компонентами](/img/props/6A.png)

- Создайте кнопку TMP.

![Окно Inspector с кнопкой и компонентами](/img/props/6B.png)

- Добавьте компонент Basis Prop, введите имя Asset Bundle и нажмите кнопку 'Create Prop Bee File'. Сгенерированные .bee-файлы будут использоваться Basis Runtime Loader для загрузки пропа в сцену. В этом примере мы снова используем синхронизированный по сети куб из предыдущего раздела.

![Окно Inspector с синхронизированным кубом и компонентами](/img/props/6B-.png)

- Создайте пустой Gameobject и прикрепите к нему компонент Runtime Loader. Он будет использоваться для появления пропа. Чтобы предотвратить генерацию пропов при запуске сцены, отключите компонент в инспекторе.

![Окно Inspector с отключённым Runtime Loader](/img/props/6C.png)

<Callout type="info">
Если вы хотите, чтобы ваш проп был синхронизирован по сети, добавьте к нему компонент сетевой синхронизации. Например, вы можете использовать компонент Basis Object Network Syncing, как показано ранее, для синхронизации движения пропа.

Вы также можете создать собственный код сетевой синхронизации. Для вдохновения и старта обратитесь к примерам пакетов *Basis Ported Pooltable* и *Basis Vehicles*, которые можно найти внутри проекта.
</Callout>

- Выберите вашу кнопку и добавьте к ней событие onClick(). Назначьте ваш GameObject Runtime Loader, убедившись, что выбран метод BasisRuntimeLoader.LoadNow, который будет вызываться при нажатии кнопки.

![Окно Inspector с событием кнопки, вызывающим RuntimeLoader](/img/props/6D.png)

- Откройте редактор Load Config.

- Введите сгенерированный пароль пропа и .bee-URL.

- Убедитесь, что выбран Persistent, чтобы синхронизированные изменения сохранялись для игроков, подключившихся позже.

- Нажмите Save XML.

![Окно Inspector с конфигурацией загрузки](/img/props/6E.png)

- Если ещё не сделали, создайте пустой Gamebject и прикрепите компонент Basis Scene, заполнив соответствующие поля. После настройки нажмите 'Create Scene Bee File'.

![Окно Inspector со сценой Basis](/img/props/6F.png)

- Откройте редактор Load Config снова.

- Введите сгенерированный пароль сцены и .bee-URL.

- Убедитесь, что выбран Persistent, чтобы синхронизированные изменения сохранялись для игроков, подключившихся позже.

- Нажмите Save XML.

![Окно Inspector с кнопкой и компонентами](/img/props/6G.png)

- Откройте сцену 'Initialization' и нажмите Play. Это инициализирует созданную сцену и сетевые объекты. В следующий раз, когда вы загрузите сцену через сервер во время работы приложения Basis, вы сможете появлять проп с помощью кнопки.

![Окно Inspector с кнопкой и компонентами](/img/props/6H.png)

<Callout type="info">
Примечание: Как локальные, так и удалённые игроки могут появлять кубы!
</Callout>
