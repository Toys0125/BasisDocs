## 소개

여기서 초점은 클라우드에서 게임 서버를 호스팅하는 다양한 방법이 아닌 Basis 서버 특정 구성에 있습니다.

서버의 멀티 스레딩은 사용 가능한 코어가 많을수록 더 잘 작동하지만 두 개의 코어에서도 잘 작동합니다. 소셜 VR은 많은 대역폭을 필요로 하므로 서비스 공급자를 선택할 때 이 점을 염두에 두세요.

이 문서는 수정되지 않은 Basis 서버에 적용됩니다: 2025년 5월 LTS 브랜치의 버전 6 {/* `commit 672035f0afe87b2fbe4e36bbd7f6cd340059d976` of lts branch */}

<Callout type="info" title="서버 버전에 대한 유용한 팁">
클론한 저장소 코드에서 **ServerVersion**을 찾을 수 있습니다.  
클라이언트와 서버 코드는 버전에 대해 **동의해야 합니다**.

```csharp
BasisNetworkCore\BasisNetworkVersion.cs
5:        public static ushort ServerVersion = 6;
```
</Callout>

## [Docker](https://docs.docker.com/get-started/docker-overview/)로 실행

[docker-compose](https://docs.docker.com/compose/)를 사용하는 것이 Basis 서버를 시작하는 가장 쉽고 빠른 방법입니다. 서버의 Docker 이미지는 `long-term-support`와 `developer` 브랜치 모두 GitHub에 각각 `latest`와 `nightly`로 게시됩니다.

```yml
services:
  basis-server:
    image: ghcr.io/basisvr/basis-server:latest # or :nightly for most recent changes
    container_name: basis-server
    init: true 					 # handle process termination
    restart: unless-stopped
    environment:
      SetPort: 4296               # basis port
      HealthCheckPort: 10666      # http healthcheck port
      PromethusPort: 1234         # exists but is unused
      Password: default_password
      PeerLimit: 1024             # 1024 is the maximum supported
      EnableStatistics: true
      EnableConsole: false
    ports:
      - "4296:4296/udp"
      - "10666:10666/tcp"
      - "1234:1234/tcp" # unused
    volumes:
      - ./initialresources:/app/initialresources:ro
      - ./config:/app/config
      - ./logs:/app/logs
    # exercise health endpoint (optional)
    healthcheck:
      # http GET request
      test: ["CMD", "curl", "-f", "http://localhost:10666/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 2s
``` 
`config.xml`을 만들기 위해 한 번 실행합니다(예에서 `./config` 아래). XML의 설정은 환경 변수로 재정의됩니다.

## Windows에서 컴파일 및 실행

저장소의 LTS 브랜치를 클론했다고 가정합니다:
`git clone -b long-term-support https://github.com/BasisVR/Basis.git`

`Basis Server` 폴더에서 Visual Studio Solution 파일(sln)을 찾을 수 있습니다.
이 단계는 Windows 11에 Microsoft Visual Studio Community 2022 (64비트) - 버전 17.12.3을 사용한다고 가정합니다.

BasisNetworkConsole은 컴파일의 대상이 되는 csproj입니다.
상단의 시작 항목 메뉴에서 이 프로젝트를 선택할 수 있습니다.
선택 후 녹색 화살표를 클릭하여 빌드합니다.

컴파일 후 `\Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0\`로 이동하여
서버 exe와 컴파일된 종속성을 찾을 수 있습니다.

Windows에서 서버를 로컬로 실행하려면 이 폴더에서 명령 프롬프트를 열고 `.\BasisNetworkConsole.exe`를 실행합니다.

`\Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0> .\BasisNetworkConsole.exe`

다음과 같은 내용을 보여주는 콘솔이 열립니다:

## Linux에서 컴파일 및 실행

### Debian/Ubuntu 요구사항

Ubuntu 22.04:

`sudo add-apt-repository ppa:dotnet/backports`

Debian 또는 기타 특이한 설정:

```bash
wget https://packages.microsoft.com/config/debian/12/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb
```

그런 다음 설치를 계속합니다:

`sudo apt-get update && sudo apt-get install -y dotnet-sdk-9.0`

### Basis 가져오기

Basis를 다운로드할 디렉토리로 이동하여 다음 명령을 실행합니다:

`git clone -b long-term-support https://github.com/BasisVR/Basis`

### Basis 서버 빌드

새 터미널을 열고 Basis 서버가 포함된 폴더로 cd합니다. 예: `cd Basis/Basis\ Server`

`dotnet restore`

그런 다음 다음으로 빌드합니다:

`dotnet build` (디버그용)

또는

`dotnet build --configuration Release` (릴리스용)

### 실행

BasisServerConsole 폴더로 이동합니다(예: `/BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`)
그리고 실행합니다:

`dotnet .\.BasisNetworkConsole.dll`

<Callout type="info" title="더 집중된 Linux 릴리스를 위해">

다음으로 컴파일할 수 있습니다:

`dotnet publish -f net9.0 --self-contained --os Linux`

`--self-contained` 스위치는 dotnet 없이 OS에서 실행할 수 있도록 추가됩니다.

이 방법으로 컴파일하면(또는 시스템에 빌드를 대상으로 설정하면) 다음을 실행할 수 있습니다:

`./BasisNetworkConsole`

`BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`에서.

</Callout>
