## Введение

Здесь основное внимание уделяется конфигурации, специфичной для сервера Basis, а не многочисленным способам хостинга игровых серверов в облаке.

Многопоточность на сервере работает лучше при большем количестве доступных ядер, но было доказано, что она хорошо работает даже с двумя ядрами. Social VR требует много полосы пропускания, поэтому учитывайте это при выборе провайдера.

Этот документ применим к немодифицированному серверу Basis: Версия 6 из ветки LTS по состоянию на май 2025 года {/* `commit 672035f0afe87b2fbe4e36bbd7f6cd340059d976` ветки lts */}

<Callout type="info" title="Полезный совет о версии сервера">
**ServerVersion** можно найти в коде, клонированном из репозитория.  
Код клиента и сервера **должен совпадать** по версии.

```csharp
BasisNetworkCore\BasisNetworkVersion.cs
5:        public static ushort ServerVersion = 6;
```
</Callout>

## Запуск с Docker

Использование docker-compose — самый простой и быстрый способ запустить сервер Basis. Docker-образы сервера публикуются на GitHub для веток `long-term-support` и `developer` под именами `latest` и `nightly` соответственно.

```yml
services:
  basis-server:
    image: ghcr.io/basisvr/basis-server:latest # or :nightly for most recent changes
    container_name: basis-server
    init: true 					 # handle process termination
    restart: unless-stopped
    environment:
      SetPort: 4296               # basis port
      HealthCheckPort: 10666      # http healthcheck port
      PromethusPort: 1234         # exists but is unused
      Password: default_password
      PeerLimit: 1024             # 1024 is the maximum supported
      EnableStatistics: true
      EnableConsole: false
    ports:
      - "4296:4296/udp"
      - "10666:10666/tcp"
      - "1234:1234/tcp" # unused
    volumes:
      - ./initialresources:/app/initialresources:ro
      - ./config:/app/config
      - ./logs:/app/logs
    # exercise health endpoint (optional)
    healthcheck:
      # http GET request
      test: ["CMD", "curl", "-f", "http://localhost:10666/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 2s
``` 
Запустите один раз для создания `config.xml` (в `./config` в примере). Настройки в XML будут переопределены переменными окружения.

## Компиляция и запуск на Windows

Предполагается, что вы клонировали LTS-ветку репозитория.
`git clone -b long-term-support https://github.com/BasisVR/Basis.git`

Файл решения Visual Studio (sln) находится в директории `Basis Server`.
Эти шаги предполагают Windows 11 с Microsoft Visual Studio Community 2022 (64-бит) - Версия 17.12.3.

BasisNetworkConsole — это csproj, который необходимо выбрать для компиляции.
Вы должны иметь возможность выбрать этот проект из меню запускаемого элемента в верхней части.
После выбора нажмите зелёную стрелку для сборки.

После компиляции перейдите в `\Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0\`,
чтобы найти exe сервера и скомпилированные зависимости.

Для локального запуска сервера на Windows откройте командную оболочку в этом каталоге и выполните `.\BasisNetworkConsole.exe`

`\Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0> .\BasisNetworkConsole.exe`

Должна открыться консоль примерно со следующим содержимым:

```txt title="Пример вывода консоли сервера"

```

При запуске демо-клиента Basis обязательно укажите `localhost`. Опция находится в расширенной панели при первом запуске клиента.

## Компиляция и запуск на Linux

### Предварительные требования для Debian/Ubuntu

Для Ubuntu 22.04:

`sudo add-apt-repository ppa:dotnet/backports`

Для Debian или других нестандартных конфигураций:

```bash
wget https://packages.microsoft.com/config/debian/12/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb
```

Затем продолжите установку:

`sudo apt-get update && sudo apt-get install -y dotnet-sdk-9.0`

### Получение Basis

Перейдите в папку, куда вы хотите загрузить Basis, и выполните следующую команду:

`git clone -b long-term-support https://github.com/BasisVR/Basis`

### Сборка сервера Basis

Откройте новый терминал и перейдите в директорию с Basis Server: `cd Basis/Basis\ Server`

`dotnet restore`

Затем выполните сборку одним из способов:

`dotnet build` (для отладки)

или

`dotnet build --configuration Release` (для релиза)

### Выполнение

Перейдите в директорию BasisServerConsole (что-то вроде `/BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`)
и запустите:

`dotnet .\.BasisNetworkConsole.dll`

<Callout type="info" title="Для более специализированного Linux-релиза">

Можно скомпилировать следующим образом:

`dotnet publish -f net9.0 --self-contained --os Linux`

Ключ `--self-contained` добавлен для возможности запуска на ОС без установленного dotnet.

Если вы компилировали этим методом (или ваша сборка ориентирована на систему), вы должны иметь возможность запустить:

`./BasisNetworkConsole`

из `BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`.

</Callout>
