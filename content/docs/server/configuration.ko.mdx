### Linux에서 컴파일 및 실행

#### Debian/Ubuntu 요구사항

Ubuntu 22.04:
`sudo add-apt-repository ppa:dotnet/backports`

Debian 또는 기타 특이한 설정:

```
wget https://packages.microsoft.com/config/debian/12/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb
```

그런 다음 설치를 계속합니다:

`sudo apt-get update && sudo apt-get install -y dotnet-sdk-9.0`

#### Basis 가져오기

Basis를 다운로드할 디렉토리로 이동하여 다음 명령을 실행합니다:

`git clone -b long-term-support https://github.com/BasisVR/Basis`

#### Basis 서버 빌드

새 터미널을 열고 Basis 서버가 포함된 폴더로 cd합니다. 예: `cd Basis/Basis\ Server`

`dotnet restore`

그런 다음 다음으로 빌드합니다:

`dotnet build` (디버그용)

또는

`dotnet build --configuration Release` (릴리스용)

#### 실행

BasisServerConsole 폴더로 이동합니다(예: `/BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`)
그리고 실행합니다:

`dotnet .\BasisNetworkConsole.dll`

## 방화벽

다음 포트를 열어야 할 수 있습니다:
 * 1234/tcp
 * 10666/tcp
 * 4296/udp

## 구성 파일

서버를 처음 실행하면 서버 exe가 있는 폴더에 여러 구성 파일과 구조가 생성됩니다.

| 파일/폴더                | 설명                                        |
|--------------------------|---------------------------------------------|
| `config.xml`             | 서버의 기본 구성 파일.                       |
| `initialresources/`      | 이 폴더에는 서버가 요청 시 로드하는 월드 및 오브젝트 파일이 포함됩니다. |
| `Log/`                   | 서버 로그가 저장되는 폴더.                   |
| `admins.xml`             | 관리자 권한이 있는 사용자 목록.             |
| `banned_players.xml`     | 차단된 플레이어 계정 목록.                  |

## Config.xml

| 변수                                | 기본값                | 설명 |
|-------------------------------------|------------------------|-------------|
| `PeerLimit`                         | 1024                   | 서버에서 허용되는 최대 플레이어 수. |
| `SetPort`                           | 4296                   | 서버가 수신 대기하는 네트워크 포트. |

| `SimulatePacketLoss`               | false                  | 테스트를 위한 패킷 손실 시뮬레이션. |

| `BSRSMillisecondDefaultInterval`  | 50                     |  |
| `BSRBaseMultiplier`               | 1                      |  |
| `BSRSIncreaseRate`                | 0.005                  |  |
| `OverrideAutoDiscoveryOfIpv`      | false                  |  |
| `IPv4Address`                      | 0.0.0.0                | IPv4 바인딩 주소. |
| `IPv6Address`                      | ::1                    | IPv6 바인딩 주소. |
| `PromethusPort`                   | 1234                   | Prometheus 통계용 포트. |
| `PromethusUrl`                    | /metrics               | Prometheus 통계용 URL 경로. |

| `UseAuth`                          | true                   | 서버 인증 활성화/비활성화. |

<small>저수준 네트워킹은 <strong>LiteNetLib</strong>라는 서드파티 라이브러리에 의해 처리됩니다. 이 코드와 관련된 구성 변수는 <code>LNL</code>로 표시됩니다.</small>

### 인증 설정

```xml
<UseAuth>true</UseAuth> 
```
인증을 완전히 비활성화하기 위한 것입니다.

```xml
<UseAuthIdentity>true</UseAuthIdentity>
```
DID 인증 공급자 활성화/비활성화.

```xml
<BasisUserRestrictionMode>Normal</BasisUserRestrictionMode>
```
아직 완료되지 않았습니다. 관리자 모더레이션.

```xml
<HowManyDuplicateAuthCanExist>2</HowManyDuplicateAuthCanExist>
```
동일한 인증 자격 증명으로 연결할 수 있는 사람 수.

## 로드 가능한 구성 파일

요청 시 로드할 월드 및 오브젝트 파일을 설정할 수 있습니다.
이것들은 `InitialResources` 폴더에 넣는 파일들입니다.

<Callout type="warn">
사용하는 외부 URL은 청킹을 지원하는 적절한 서버에서 제공되어야 합니다.
모든 최신 서버는 이를 지원해야 하지만, 간단한 로컬 개발 서버는 지원하지 않을 수 있습니다.
</Callout>

<Callout type="info" title="무언가 로드하기">
새 씬을 테스트하는 한 가지 방법은 Unity에서 씬을 만들고 BEE 파일로 내보내는 것입니다. 해당 BEE 파일을 HTTP를 지원하는 서버에 업로드한 후, URL을 `CombinedURL`에 입력하고 `dontuploadmepassword.txt`에서 비밀번호를 가져와 `UnlockPassword`에 입력합니다.
</Callout>

| 변수             | 정의 |
|------------------|------------|
| `Mode`           | 구성 모드: `0` = 게임 오브젝트, `1` = 씬. |
| `LoadedNetID`    | 네트워크 ID: 서버는 클라이언트가 이 값을 참조로 쿼리하고 스폰할 수 있는 네트워크 오브젝트 목록을 유지합니다. |
| `UnlockPassword` | 리소스를 잠금 해제하거나 접근하는 데 필요한 비밀번호. |
| `CombinedURL`    | `.BEE` 파일의 URL |
| `IsLocalLoad`    | 리소스가 로컬(`true`)에서 로드되는지 인터넷(`false`)에서 로드되는지 나타냅니다. |
| `PositionX`      | 오브젝트의 X축 월드 위치. |
| `PositionY`      | 오브젝트의 Y축 월드 위치. |
| `PositionZ`      | 오브젝트의 Z축 월드 위치. |
| `QuaternionX`    | 오브젝트 회전의 X 성분(쿼터니언). |
| `QuaternionY`    | 오브젝트 회전의 Y 성분(쿼터니언). |
| `QuaternionZ`    | 오브젝트 회전의 Z 성분(쿼터니언). |
| `QuaternionW`    | 오브젝트 회전의 W 성분(쿼터니언). |
| `ScaleX`         | X축 스케일 팩터. |
| `ScaleY`         | Y축 스케일 팩터. |
| `ScaleZ`         | Z축 스케일 팩터. |
| `Persist`        | 오브젝트를 스폰한 플레이어가 서버를 떠나도 오브젝트가 계속 존재하나요? |

### 예시 구성

```xml title="BasisLoadableConfiguration-예시"
<BasisLoadableConfiguration>

    <!-- Mode of the configuration: Mode  0 = Game object, 1 = Scene -->
    <Mode>0</Mode>

    <!-- A Network ID: Server keeps a list of Network Objects that Clients can query and spawn with this value. -->
    <LoadedNetID></LoadedNetID>

    <!-- Unlock password -->
    <UnlockPassword>ffee4439eff</UnlockPassword>

    <!-- Combined URL link to BEE file -->
    <CombinedURL>https://www.example.com/2lkj23j.BEE</CombinedURL>

    <!-- Does this resource exist on the internet or in the client? -->
    <IsLocalLoad>false</IsLocalLoad>

    <!-- Position values -->
    <PositionX>0</PositionX>
    <PositionY>0</PositionY>
    <PositionZ>0</PositionZ>

    <!-- Quaternion values -->
    <QuaternionX>0</QuaternionX>
    <QuaternionY>0</QuaternionY>
    <QuaternionZ>0</QuaternionZ>
    <QuaternionW>1</QuaternionW>

    <!-- Scale values -->
    <ScaleX>1</ScaleX>
    <ScaleY>1</ScaleY>
    <ScaleZ>1</ScaleZ>

    <!--  If the player that spawned the object leaves the server does the object continue to exist?  -->
    <Persist>false</Persist>

</BasisLoadableConfiguration>
```
