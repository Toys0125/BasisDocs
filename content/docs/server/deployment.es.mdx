## Introducción

El enfoque aquí es la configuración específica del servidor Basis y no las muchas formas de alojar servidores de juegos en la nube.

El multithreading en el servidor funciona mejor cuantos más núcleos estén disponibles, pero se ha demostrado que funciona bien con tan solo dos núcleos. El VR social exige mucho ancho de banda, así que tenga eso en cuenta al elegir su proveedor de servicios.

Este documento se aplica a un servidor Basis sin modificar: Versión 6 de la rama LTS a partir de mayo de 2025 {/* `commit 672035f0afe87b2fbe4e36bbd7f6cd340059d976` de la rama lts */}

<Callout type="info" title="Consejo útil sobre la versión del servidor">
Puede encontrar el **ServerVersion** en el código que clonó del repositorio.  
El código del cliente y el servidor **deben coincidir** en la versión.

```csharp
BasisNetworkCore\BasisNetworkVersion.cs
5:        public static ushort ServerVersion = 6;
```
</Callout>

## Ejecutar con [Docker](https://docs.docker.com/get-started/docker-overview/)

Usar [docker-compose](https://docs.docker.com/compose/) es la forma más fácil y rápida de poner en marcha un servidor Basis. Las imágenes Docker del servidor se publican en GitHub tanto en las ramas `long-term-support` como `developer`, bajo `latest` y `nightly` respectivamente.

```yml
services:
  basis-server:
    image: ghcr.io/basisvr/basis-server:latest # or :nightly for most recent changes
    container_name: basis-server
    init: true 					 # handle process termination
    restart: unless-stopped
    environment:
      SetPort: 4296               # basis port
      HealthCheckPort: 10666      # http healthcheck port
      PromethusPort: 1234         # exists but is unused
      Password: default_password
      PeerLimit: 1024             # 1024 is the maximum supported
      EnableStatistics: true
      EnableConsole: false
    ports:
      - "4296:4296/udp"
      - "10666:10666/tcp"
      - "1234:1234/tcp" # unused
    volumes:
      - ./initialresources:/app/initialresources:ro
      - ./config:/app/config
      - ./logs:/app/logs
    # exercise health endpoint (optional)
    healthcheck:
      # http GET request
      test: ["CMD", "curl", "-f", "http://localhost:10666/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 2s
``` 
Ejecute una vez para crear `config.xml` (bajo `./config` en el ejemplo). Los ajustes en XML serán reemplazados por variables de entorno.

## Compilar y ejecutar en Windows

Asumiendo que ha clonado la rama LTS del repositorio.
`git clone -b long-term-support https://github.com/BasisVR/Basis.git`

Encontrará el archivo de solución de Visual Studio (sln) en el directorio `Basis Server`.
Estos pasos asumen Windows 11 con Microsoft Visual Studio Community 2022 (64 bits) - Versión 17.12.3.

BasisNetworkConsole es el csproj que necesitará seleccionar para compilar.
Debería poder seleccionar ese proyecto desde el menú de elemento de inicio en la parte superior.
Una vez seleccionado, haga clic en la flecha verde para compilar.

Una vez compilado, navegue a `\Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0\`
para encontrar el exe del servidor y las dependencias compiladas.

Para ejecutar el servidor localmente en Windows, abra una shell de comandos en este directorio y ejecute `.\BasisNetworkConsole.exe`

`\Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0> .\BasisNetworkConsole.exe`

Esto debería abrir una consola y mostrar algo como lo siguiente:


## Compilar y ejecutar en Linux

### Prerrequisitos para Debian/Ubuntu

Para Ubuntu 22.04:

`sudo add-apt-repository ppa:dotnet/backports`

Para Debian u otras configuraciones peculiares:

```bash
wget https://packages.microsoft.com/config/debian/12/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb
```

Luego continúe la instalación:

`sudo apt-get update && sudo apt-get install -y dotnet-sdk-9.0`

### Obtener Basis

Cambie a la carpeta donde desea descargar Basis y ejecute el siguiente comando:

`git clone -b long-term-support https://github.com/BasisVR/Basis`

### Compilar el servidor Basis

Abra una nueva terminal y acceda al directorio con Basis Server: `cd Basis/Basis\ Server`

`dotnet restore`

Luego compile con:

`dotnet build` (para depuración)

o

`dotnet build --configuration Release` (para release)

### Ejecutar

Navegue al directorio BasisServerConsole (algo como `/BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`)
y ejecute:

`dotnet .\.BasisNetworkConsole.dll`

<Callout type="info" title="Para un release de Linux más específico">

Puede compilar con lo siguiente:

`dotnet publish -f net9.0 --self-contained --os Linux`

El modificador `--self-contained` se agrega para permitir la ejecución en un SO sin dotnet instalado.

Si compila mediante este método (o su compilación apuntó al sistema), debería poder ejecutar:

`./BasisNetworkConsole`

desde `BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`.

</Callout>
