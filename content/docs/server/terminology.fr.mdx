## Termes clés

**Joueur local**  
Les données d'avatar local qui vous représentent _vous_ (l'utilisateur ou le client).

**Joueur distant**  
Représente les autres joueurs dont les données vous sont envoyées par le serveur.

**Client Basis**  
L'application que vous écrivez et qui communiquera avec le serveur.

**Données de mouvement**  
Les données représentant les positions des os du joueur ainsi que sa position et sa rotation dans le monde.
Stockées dans un format sérialisable réseau dans la structure de données LASM.

**LASM**  
Signifie Local Avatar Sync Message, la structure de données contenant des données compressées
sur les muscles, la position et la rotation d'un joueur qui sont envoyées sur le réseau.
