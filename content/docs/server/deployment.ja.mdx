## はじめに

ここではBasisサーバー固有の設定に焦点を当てており、クラウドでゲームサーバーをホストする多くの方法については説明しません。

サーバーのマルチスレッドは利用可能なコア数が多いほど最高に動作しますが、2コアという少ないコア数でも良好に動作することが示されています。ソーシャルVRは大量の帯域幅を必要とするため、サービスプロバイダーを選ぶ際にその点を念頭に置いてください。

このドキュメントは、2025年5月時点のLTSブランチの未変更Basisサーバー：バージョン6に適用されます {/* `commit 672035f0afe87b2fbe4e36bbd7f6cd340059d976` of lts branch */}

<Callout type="info" title="サーバーバージョンに関するヒント">
リポジトリからクローンしたコードで**ServerVersion**を確認できます。  
クライアントとサーバーのコードはバージョンが**一致**している必要があります。

```csharp
BasisNetworkCore\BasisNetworkVersion.cs
5:        public static ushort ServerVersion = 6;
```
</Callout>

## [Docker](https://docs.docker.com/get-started/docker-overview/)で実行する

[docker-compose](https://docs.docker.com/compose/)を使用するのが、Basisサーバーを起動する最も簡単で速い方法です。サーバーのDockerイメージはGitHubに`long-term-support`と`developer`ブランチの両方で、それぞれ`latest`と`nightly`として公開されています。

```yml
services:
  basis-server:
    image: ghcr.io/basisvr/basis-server:latest # or :nightly for most recent changes
    container_name: basis-server
    init: true 					 # handle process termination
    restart: unless-stopped
    environment:
      SetPort: 4296               # basis port
      HealthCheckPort: 10666      # http healthcheck port
      PromethusPort: 1234         # exists but is unused
      Password: default_password
      PeerLimit: 1024             # 1024 is the maximum supported
      EnableStatistics: true
      EnableConsole: false
    ports:
      - "4296:4296/udp"
      - "10666:10666/tcp"
      - "1234:1234/tcp" # unused
    volumes:
      - ./initialresources:/app/initialresources:ro
      - ./config:/app/config
      - ./logs:/app/logs
    # exercise health endpoint (optional)
    healthcheck:
      # http GET request
      test: ["CMD", "curl", "-f", "http://localhost:10666/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 2s
``` 
`config.xml`を作成するために一度実行します（例では`./config`の下）。XMLの設定は環境変数によって上書きされます。

## Windowsでコンパイルして実行する

リポジトリのLTSブランチをクローンしたと仮定しています。
`git clone -b long-term-support https://github.com/BasisVR/Basis.git`

Visual Studioのソリューションファイル（sln）は`Basis Server`ディレクトリにあります。
これらの手順はWindows 11とMicrosoft Visual Studio Community 2022（64ビット）- バージョン17.12.3を前提とします。

BasisNetworkConsoleは、コンパイルのためにターゲットにする必要があるcsprojです。
上部のスタートアップアイテムメニューからそのプロジェクトを選択できるはずです。
選択後、緑の矢印をクリックしてビルドします。

コンパイル後、`\Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0\`に移動して
サーバーのexeとコンパイルされた依存関係を見つけます。

Windowsでサーバーをローカルで実行するには、このディレクトリでコマンドシェルを開いて`.\BasisNetworkConsole.exe`を実行します。

`\Basis\Basis Server\BasisServerConsole\bin\Debug\net9.0> .\BasisNetworkConsole.exe`

これでコンソールが開き、次のような内容が表示されます：


## Linuxでコンパイルして実行する

### Debian/Ubuntuの前提条件

Ubuntu 22.04の場合：

`sudo add-apt-repository ppa:dotnet/backports`

Debianまたはその他の特殊なセットアップの場合：

```bash
wget https://packages.microsoft.com/config/debian/12/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb
```

その後インストールを続けます：

`sudo apt-get update && sudo apt-get install -y dotnet-sdk-9.0`

### Basisを取得する

Basisをダウンロードしたいフォルダに移動し、以下のコマンドを実行します：

`git clone -b long-term-support https://github.com/BasisVR/Basis`

### Basisサーバーをビルドする

新しいターミナルを開き、Basis Serverのディレクトリに移動します：`cd Basis/Basis\ Server`

`dotnet restore`

その後、以下のいずれかでビルドします：

`dotnet build`（デバッグ用）

または

`dotnet build --configuration Release`（リリース用）

### 実行する

BasisServerConsoleディレクトリに移動します（`/BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`のようなパス）
そして実行します：

`dotnet .\.BasisNetworkConsole.dll`

<Callout type="info" title="より特定的なLinuxリリースの場合">

以下でコンパイルできます：

`dotnet publish -f net9.0 --self-contained --os Linux`

`--self-contained`スイッチはdotnetがインストールされていないOSでの実行を可能にするために追加されます。

この方法でコンパイルした場合（またはビルドがシステムをターゲットにしていた場合）、次を実行できるはずです：

`./BasisNetworkConsole`

`BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`から。

</Callout>

## ファイアウォール

以下のポートを開く必要があるかもしれません：
 * 1234/tcp
 * 10666/tcp
 * 4296/udp
