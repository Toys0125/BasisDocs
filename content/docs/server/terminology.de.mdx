## Schlüsselbegriffe

**Lokaler Spieler**  
Die lokalen Avatar-Daten, die _Sie_ (den Benutzer oder Client) repräsentieren.

**Entfernter Spieler**  
Repräsentiert andere Spieler, deren Daten Ihnen vom Server gesendet werden.

**Basis-Client**  
Die Anwendung, die Sie schreiben und die mit dem Server kommuniziert.

**Bewegungsdaten**  
Die Daten, die die Knochenpositionen des Spielers sowie seine Position und Rotation in der Welt darstellen.
Gespeichert in einem netzwerkserialisierbaren Format in der LASM-Datenstruktur.

**LASM**  
Steht für Local Avatar Sync Message, die Datenstruktur mit komprimierten Daten
zu Muskeln, Position und Rotation eines Spielers, die über das Netzwerk gesendet werden.
