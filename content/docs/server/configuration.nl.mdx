### Compileren en uitvoeren op Linux

#### Vereisten voor Debian/Ubuntu

Voor Ubuntu 22.04,
`sudo add-apt-repository ppa:dotnet/backports`

Voor Debian of andere eigenzinnige setups:

```
wget https://packages.microsoft.com/config/debian/12/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb
```

Ga daarna verder met de installatie:

`sudo apt-get update && sudo apt-get install -y dotnet-sdk-9.0`

#### Basis ophalen

Navigeer naar de map waarnaar je Basis wilt downloaden en voer de volgende opdracht uit:

`git clone -b long-term-support https://github.com/BasisVR/Basis`

#### Basis Server bouwen

Open een nieuwe terminal en cd naar de map met de Basis Server, bijv. `cd Basis/Basis\ Server`

`dotnet restore`

Bouw daarna met:

`dotnet build` (voor debug)

of

`dotnet build --configuration Release` (voor release)

#### Uitvoeren

Navigeer naar de BasisServerConsole-map (zoiets als `/BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`)
en voer uit:

`dotnet .\BasisNetworkConsole.dll`

## Firewall

Mogelijk wil je de volgende poorten openen:
 * 1234/tcp
 * 10666/tcp
 * 4296/udp

## Configuratiebestanden

De eerste keer dat je de server uitvoert, worden er verschillende configuratiebestanden en een structuur aangemaakt in de map waar de server exe zich bevindt.

| Bestand/Map              | Beschrijving                                |
|--------------------------|---------------------------------------------|
| `config.xml`             | Hoofdconfiguratiebestand voor de server.    |
| `initialresources/`      | Deze map bevat wereld- en objectbestanden die de server op verzoek laadt. |
| `Log/`                   | Map waar serverlogboeken worden opgeslagen. |
| `admins.xml`             | Lijst van gebruikers met beheerdersrechten. |
| `banned_players.xml`     | Lijst van verbannen spelersaccounts.        |

### Authenticatie-instellingen

```xml
<UseAuth>true</UseAuth> 
```
Bedoeld voor het volledig uitschakelen van auth.

```xml
<UseAuthIdentity>true</UseAuthIdentity>
```
DID-auth-provider in- of uitschakelen.

```xml
<BasisUserRestrictionMode>Normal</BasisUserRestrictionMode>
```
Nog niet voltooid. Beheerdersmatiging.

```xml
<HowManyDuplicateAuthCanExist>2</HowManyDuplicateAuthCanExist>
```
Hoeveel mensen verbinding kunnen maken met dezelfde auth-gegevens.

## Laadbare configuratiebestanden

Je kunt wereld- en objectbestanden instellen om op aanvraag te laden.
Dit zijn de bestanden die je in de map `InitialResources` plaatst.

<Callout type="warn">
Let op dat de externe URL's die je gebruikt, moeten worden geleverd door een juiste server die chunking ondersteunt.
Elke moderne server zou dit moeten doen, maar eenvoudige lokale dev-servers mogelijk niet.
</Callout>

<Callout type="info" title="Iets uploaden">
Een manier om een nieuwe scène te testen is een scène in Unity te maken, deze te exporteren als een BEE-bestand. Upload dat BEE-bestand naar een server die HTTP ondersteunt, plaats vervolgens de URL in `CombinedURL` en haal het wachtwoord op uit `dontuploadmepassword.txt` en plaats het in `UnlockPassword`.
</Callout>

| Variabele        | Definitie |
|------------------|------------|
| `Mode`           | Modus van de configuratie: `0` = Gameobject, `1` = Scène. |
| `LoadedNetID`    | Een netwerk-ID: server houdt een lijst bij van netwerkapparaten die clients kunnen opvragen en spawnen met deze waarde als referentie. |
| `UnlockPassword` | Wachtwoord vereist om de bron te ontgrendelen of toegang te krijgen. |
| `CombinedURL`    | URL naar het `.BEE`-bestand |
| `IsLocalLoad`    | Geeft aan of de bron lokaal wordt geladen (`true`) of van het internet (`false`). |
| `PositionX`      | X-as wereldpositie van het object. |
| `PositionY`      | Y-as wereldpositie van het object. |
| `PositionZ`      | Z-as wereldpositie van het object. |
| `QuaternionX`    | X-component van de rotatie van het object (quaternion). |
| `QuaternionY`    | Y-component van de rotatie van het object (quaternion). |
| `QuaternionZ`    | Z-component van de rotatie van het object (quaternion). |
| `QuaternionW`    | W-component van de rotatie van het object (quaternion). |
| `ScaleX`         | X-as schaalfactor. |
| `ScaleY`         | Y-as schaalfactor. |
| `ScaleZ`         | Z-as schaalfactor. |
| `Persist`        | Als de speler die het object heeft gespawnd de server verlaat, blijft het object dan bestaan? |

### Voorbeeldconfiguratie

```xml title="BasisLoadableConfiguration-voorbeeld"
<BasisLoadableConfiguration>

    <!-- Mode of the configuration: Mode  0 = Game object, 1 = Scene -->
    <Mode>0</Mode>

    <!-- A Network ID: Server keeps a list of Network Objects that Clients can query and spawn with this value. -->
    <LoadedNetID></LoadedNetID>

    <!-- Unlock password -->
    <UnlockPassword>ffee4439eff</UnlockPassword>

    <!-- Combined URL link to BEE file -->
    <CombinedURL>https://www.example.com/2lkj23j.BEE</CombinedURL>

    <!-- Does this resource exist on the internet or in the client? -->
    <IsLocalLoad>false</IsLocalLoad>

    <!-- Position values -->
    <PositionX>0</PositionX>
    <PositionY>0</PositionY>
    <PositionZ>0</PositionZ>

    <!-- Quaternion values -->
    <QuaternionX>0</QuaternionX>
    <QuaternionY>0</QuaternionY>
    <QuaternionZ>0</QuaternionZ>
    <QuaternionW>1</QuaternionW>

    <!-- Scale values -->
    <ScaleX>1</ScaleX>
    <ScaleY>1</ScaleY>
    <ScaleZ>1</ScaleZ>

    <!--  If the player that spawned the object leaves the server does the object continue to exist?  -->
    <Persist>false</Persist>

</BasisLoadableConfiguration>
```
