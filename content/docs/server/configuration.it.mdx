### Compilazione ed esecuzione su Linux

#### Requisiti per Debian/Ubuntu

Per Ubuntu 22.04,
`sudo add-apt-repository ppa:dotnet/backports`

Per Debian o altre configurazioni particolari:

```
wget https://packages.microsoft.com/config/debian/12/packages-microsoft-prod.deb -O packages-microsoft-prod.deb
sudo dpkg -i packages-microsoft-prod.deb
rm packages-microsoft-prod.deb
```

Poi continua con l'installazione:

`sudo apt-get update && sudo apt-get install -y dotnet-sdk-9.0`

#### Ottenere Basis

Vai alla directory in cui vuoi scaricare Basis ed esegui il seguente comando:

`git clone -b long-term-support https://github.com/BasisVR/Basis`

#### Compilare il Basis Server

Apri un nuovo terminale e vai alla cartella contenente il Basis Server, ad es. `cd Basis/Basis\ Server`

`dotnet restore`

Poi compila con:

`dotnet build` (per debug)

oppure

`dotnet build --configuration Release` (per release)

#### Esecuzione

Vai alla cartella BasisServerConsole (qualcosa come `/BasisServerConsole/bin/Debug/net9.0/BasisNetworkConsole`)
ed esegui:

`dotnet .\BasisNetworkConsole.dll`

## Firewall

Potresti voler aprire le seguenti porte:
 * 1234/tcp
 * 10666/tcp
 * 4296/udp

## File di configurazione

La prima volta che esegui il server, vengono creati vari file di configurazione e una struttura nella cartella dove si trova l'exe del server.

| File/Cartella            | Descrizione                                 |
|--------------------------|---------------------------------------------|
| `config.xml`             | File di configurazione principale del server. |
| `initialresources/`      | Questa cartella contiene file di mondo e oggetti che il server carica su richiesta. |
| `Log/`                   | Cartella dove vengono salvati i log del server. |
| `admins.xml`             | Elenco degli utenti con privilegi di amministratore. |
| `banned_players.xml`     | Elenco degli account giocatori bannati. |

### Impostazioni di autenticazione

```xml
<UseAuth>true</UseAuth> 
```
Destinato alla disabilitazione completa dell'autenticazione.

```xml
<UseAuthIdentity>true</UseAuthIdentity>
```
Abilitare o disabilitare il provider di autenticazione DID.

```xml
<BasisUserRestrictionMode>Normal</BasisUserRestrictionMode>
```
Non ancora completato. Moderazione degli amministratori.

```xml
<HowManyDuplicateAuthCanExist>2</HowManyDuplicateAuthCanExist>
```
Quante persone possono connettersi con le stesse credenziali di autenticazione.

## File di configurazione caricabili

Puoi impostare file di mondo e oggetti da caricare su richiesta.
Questi sono i file che metti nella cartella `InitialResources`.

<Callout type="warn">
Tieni presente che gli URL esterni che utilizzi devono essere forniti da un server adeguato che supporti il chunking.
Qualsiasi server moderno dovrebbe farlo, ma i semplici server di sviluppo locali potrebbero non farlo.
</Callout>

<Callout type="info" title="Caricare qualcosa">
Un modo per testare una nuova scena è creare una scena in Unity, esportarla come file BEE. Carica quel file BEE su un server che supporta HTTP, poi inserisci l'URL in `CombinedURL` e recupera la password da `dontuploadmepassword.txt` e inseriscila in `UnlockPassword`.
</Callout>

| Variabile        | Definizione |
|------------------|------------|
| `Mode`           | Modalità della configurazione: `0` = GameObject, `1` = Scena. |
| `LoadedNetID`    | Un ID di rete: il server mantiene un elenco di oggetti di rete che i client possono interrogare e spawnare con questo valore come riferimento. |
| `UnlockPassword` | Password richiesta per sbloccare o accedere alla risorsa. |
| `CombinedURL`    | URL al file `.BEE` |
| `IsLocalLoad`    | Indica se la risorsa viene caricata localmente (`true`) o da internet (`false`). |
| `PositionX`      | Posizione mondiale sull'asse X dell'oggetto. |
| `PositionY`      | Posizione mondiale sull'asse Y dell'oggetto. |
| `PositionZ`      | Posizione mondiale sull'asse Z dell'oggetto. |
| `QuaternionX`    | Componente X della rotazione dell'oggetto (quaternione). |
| `QuaternionY`    | Componente Y della rotazione dell'oggetto (quaternione). |
| `QuaternionZ`    | Componente Z della rotazione dell'oggetto (quaternione). |
| `QuaternionW`    | Componente W della rotazione dell'oggetto (quaternione). |
| `ScaleX`         | Fattore di scala sull'asse X. |
| `ScaleY`         | Fattore di scala sull'asse Y. |
| `ScaleZ`         | Fattore di scala sull'asse Z. |
| `Persist`        | Se il giocatore che ha spawnato l'oggetto lascia il server, l'oggetto continua a esistere? |

### Configurazione di esempio

```xml title="BasisLoadableConfiguration-esempio"
<BasisLoadableConfiguration>

    <!-- Mode of the configuration: Mode  0 = Game object, 1 = Scene -->
    <Mode>0</Mode>

    <!-- A Network ID: Server keeps a list of Network Objects that Clients can query and spawn with this value. -->
    <LoadedNetID></LoadedNetID>

    <!-- Unlock password -->
    <UnlockPassword>ffee4439eff</UnlockPassword>

    <!-- Combined URL link to BEE file -->
    <CombinedURL>https://www.example.com/2lkj23j.BEE</CombinedURL>

    <!-- Does this resource exist on the internet or in the client? -->
    <IsLocalLoad>false</IsLocalLoad>

    <!-- Position values -->
    <PositionX>0</PositionX>
    <PositionY>0</PositionY>
    <PositionZ>0</PositionZ>

    <!-- Quaternion values -->
    <QuaternionX>0</QuaternionX>
    <QuaternionY>0</QuaternionY>
    <QuaternionZ>0</QuaternionZ>
    <QuaternionW>1</QuaternionW>

    <!-- Scale values -->
    <ScaleX>1</ScaleX>
    <ScaleY>1</ScaleY>
    <ScaleZ>1</ScaleZ>

    <!--  If the player that spawned the object leaves the server does the object continue to exist?  -->
    <Persist>false</Persist>

</BasisLoadableConfiguration>
```
