## Preguntas frecuentes

### ¿Qué es BasisVR?

BasisVR es un framework de software que empodera a los desarrolladores con integraciones tecnológicas profundas y enfoques de primer nivel para crear experiencias sociales en Realidad Virtual.

<Callout type="info">
Nota: ***Basis no es una plataforma Social VR***. Incluye primitivas útiles, pero el proyecto principal no está destinado a crear el tipo de infraestructura social escalable que requieren las plataformas Social VR. No obstaculiza específicamente a los creadores que deseen crear la suya propia, pero el proyecto en sí incluye frameworks y demos.
</Callout>

### ¿Cuáles son los requisitos del sistema?

BasisVR requiere:
- Un PC compatible con VR o un headset VR independiente
- Unity para el desarrollo
- Controladores VR compatibles

### ¿Cómo empiezo?

1. Instalar BasisVR
2. Configurar tu avatar
3. Crear o unirte a un mundo
4. ¡Empezar a explorar!

Consulta la [Introducción](/docs) para obtener instrucciones detalladas de configuración.

### ¿Puedo crear mis propios avatares?

¡Sí! BasisVR admite avatares personalizados. Consulta la [Guía de Avatar](/es/docs/avatar) para más información sobre cómo crear e importar avatares personalizados.

### ¿Cómo creo mi propio mundo?

Puedes crear mundos personalizados usando Unity. Consulta la [Guía de creación de mundos](/es/docs/world) para instrucciones paso a paso.

### ¿Cuál es la diferencia entre las contribuciones de componentes principales y las contribuciones de complementos?
R: Los componentes principales incluyen sistemas que conforman las características clave de Basis. Las contribuciones (o módulos) incluyen características adicionales que, si bien son útiles, no son requeridas por defecto. Por lo tanto, las contribuciones pueden proporcionar características fuera del alcance de la oferta principal de Basis, lo que puede ser valioso para los desarrolladores que usan Basis.

### ¿Compartir archivos BEE compromete las credenciales del servidor de alojamiento de contenido remoto?

¡No! Puedes compartir tus archivos BEE con tus amigos, por ejemplo. Un archivo BEE incluye el asset de Basis (incluso para múltiples plataformas), excluyendo la clave de cifrado utilizada para descifrarlo. En otras palabras, es básicamente un multi-asset-bundle. Si compartes la contraseña de un archivo BEE, esto permite que cualquiera que la tenga pueda acceder a todo lo que está dentro del archivo BEE (es decir, el UGC específico como accesorios, avatares, mundos).

## ¿Cómo puedo conectarme a un servidor local desde diferentes dispositivos?

Hay varias opciones para conectarse a un servidor local desde diferentes dispositivos. Por ejemplo, puedes habilitar el reenvío de puertos en tu router. Alternativamente, si prefieres no configurar tu router, puedes instalar [`Tailscale`](https://tailscale.com/download), que es muy fácil de configurar.

## ¿Dónde puedo configurar un servidor remoto?

Hay una variedad de servicios en la nube donde puedes alojar tus servidores. Ejemplos: Bunny, DigitalOcean, CloudFlare, AWS, Microsoft Azure, Google Cloud y más.
